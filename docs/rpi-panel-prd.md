# RPi Panel — Product Requirements Document
**v1.2 · Март 2026**

---

## 1. Обзор продукта

RPi Panel — лёгкая веб-панель управления для Raspberry Pi Zero W, написанная на Go. Позволяет управлять VPN-подключениями (VLESS/xray), WiFi-сетями и системными сервисами через браузер без необходимости SSH-доступа.

Панель запускается локально на RPi Zero W и доступна из локальной сети по адресу `http://192.168.1.x:8080`. Деплой осуществляется кросс-компиляцией с macOS и копированием бинарника по SCP.

---

## 2. Первый запуск (Onboarding Flow)

При первом включении RPi пользователь проходит пошаговый онбординг. Цель — минимальное трение: подключился к открытой AP, открыл панель, настроил пароли за 1 минуту.

```
Подключился к открытой AP "RPi-Panel"
        ↓
Открыл браузер → редирект на /setup
        ↓
Шаг 1: установил пароль панели (обязательно)
        ↓
Шаг 2: установил пароль AP (опционально, можно пропустить)
        ↓
Шаг 3: готово → /login → в панель
```

### 2.1 Шаг 1 — Открытая AP (из коробки)

- При первом запуске `hostapd.conf` не содержит `wpa_passphrase` — AP открытая
- SSID по умолчанию: `RPi-Panel`
- Любое устройство может подключиться без пароля
- Состояние определяется флагом `setup_done: false` в `/etc/rpi-panel/config.json`

### 2.2 Шаг 2 — Первый вход в панель

- При `setup_done: false` — любой `GET /` редиректит на `/setup`
- Страница `/setup` не требует авторизации
- Отображается приветственный экран: «Добро пожаловать в RPi Panel»

### 2.3 Шаг 3 — Мастер настройки (Setup Wizard)

Трёхшаговый wizard на одной странице, шаги переключаются через HTMX:

| Шаг | Описание |
|-----|----------|
| Шаг 1 — Пароль панели | Ввод пароля + подтверждение. Минимум 6 символов. Обязательный — нельзя пропустить. |
| Шаг 2 — Пароль AP | Ввод пароля WiFi точки доступа. Минимум 8 символов (WPA2). Можно пропустить — AP останется открытой. |
| Шаг 3 — Готово | Сводка: пароль панели установлен, статус AP (защищена / открытая). Кнопка «Войти в панель». |

### 2.4 Логика завершения онбординга

- После Шага 1: сохраняется bcrypt-хэш пароля в `config.json`
- После Шага 2 (заполнен): `wpa_passphrase` добавляется в `hostapd.conf`, рестарт hostapd
- После Шага 2 (пропущен): AP остаётся открытой, `ap_secured: false`
- После Шага 3: `setup_done: true` → все следующие запросы идут через `/login`
- Повторный онбординг невозможен пока `setup_done: true`

### 2.5 Изменение паролей после онбординга

- Пароль панели: Advanced Mode → Настройки → Сменить пароль панели
- Пароль AP: Advanced Mode → Настройки → WiFi AP → Сменить пароль / Сделать открытой
- Оба действия требуют текущий пароль панели для подтверждения

### 2.6 Состояния системы

| Состояние | Поведение |
|-----------|-----------|
| `setup_done: false`, `ap_secured: false` | Первый запуск — открытая AP, редирект на `/setup` |
| `setup_done: true`, `ap_secured: false` | Панель настроена, AP открытая (пользователь пропустил шаг 2) |
| `setup_done: true`, `ap_secured: true` | Полностью настроено — обычная работа |

## 3. Режимы интерфейса

Панель поддерживает два режима отображения. Один аккаунт — тоггл в шапке скрывает или показывает расширенный UI. Режим сохраняется в `localStorage` браузера.

### 3.1 Simple Mode (по умолчанию)

| Элемент | Описание |
|---------|----------|
| Статус VPN | Индикатор подключён/отключён + текущий внешний IP |
| Выбор сервера | Дропдаун со списком серверов, кнопка переключить |
| WiFi | Текущая сеть + кнопка сменить |
| Быстрые действия | Перезапустить VPN, обновить подписку |

Всё что не нужно обычному пользователю скрыто: логи, raw конфиги, системные сервисы, HWID, добавление серверов вручную.

### 3.2 Advanced Mode

| Секция | Содержимое |
|--------|------------|
| Дашборд | Статус всех сервисов, внешний и локальный IP |
| VLESS серверы | Список, добавление, удаление, активация |
| Подписки | Sub-ссылки Remnawave, HWID, обновление |
| WiFi | Сканирование, подключение, уровень сигнала |
| Сервисы | Рестарт каждого сервиса |
| Логи | Последние 100 строк журнала |
| Настройки | Смена пароля панели, управление паролем AP |

### 3.3 Переключение режимов

- Тоггл «Simple / Advanced» в правом углу шапки, всегда виден
- Состояние сохраняется в `localStorage`
- Реализация: CSS класс `advanced-only`, переключение на клиенте без серверной логики
- По умолчанию при первом входе — Simple Mode

```html
<!-- Пример разметки -->
<div class="advanced-only">
  <!-- логи, raw конфиги, системные сервисы -->
</div>
```

## 4. Цели и ограничения

### 4.1 Цели

- Нулевая конфигурация из коробки — подключился и сразу в панели
- Управление VLESS-серверами без ручного редактирования `config.json`
- Быстрое переключение WiFi-сетей без SSH
- Поддержка sub-ссылок Remnawave с HWID-идентификацией
- Минимальное потребление ресурсов (RPi Zero W, 512MB RAM, ARMv6)
- Простой интерфейс для нетехнических пользователей

### 4.2 Ограничения

- Только локальный доступ (LAN), без внешнего доступа из интернета
- Один аккаунт — аутентификация логин + пароль
- Нет мобильного приложения — только веб-интерфейс
- Не поддерживается VMess/Trojan в v1.0

## 5. Технический стек

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| Backend | Go (net/http) | Лёгкий, кросс-компиляция под ARMv6 |
| Frontend | HTMX + Go templates | Нет сборки, всё в одном бинарнике |
| Стили | Tailwind CDN | Не нужен npm build |
| Режимы UI | CSS + localStorage | Simple/Advanced без серверной логики |
| Онбординг | Go template + HTMX steps | Wizard без JS фреймворка |
| Хранилище | JSON файлы | Нет БД, минимальные зависимости |
| Деплой | SCP + systemd | Кросс-компиляция mac → ARMv6 |
| VPN | xray-core (VLESS) | Уже установлен на RPi |
| Проксирование | redsocks + iptables | Прозрачный proxy для AP |

## 6. Архитектура

```
Телефон → uap0 (AP) → redsocks:12345 → xray:1080 → VLESS → Интернет
```

RPi Panel управляет конфигурацией xray (`/usr/local/etc/xray/config.json`), WiFi (`wpa_supplicant`) и AP (`hostapd.conf`) через системные команды с sudo.

### Структура проекта

```
rpi-panel/
├── main.go          # HTTP сервер, роутинг, middleware авторизации и онбординга
├── setup.go         # логика Setup Wizard, запись паролей, настройка AP
├── services.go      # systemctl статус/рестарт, IP
├── vless.go         # парсинг vless://, управление xray config
├── wifi.go          # сканирование, wpa_supplicant
├── sub.go           # sub-ссылки, HWID
├── ap.go            # управление hostapd.conf, пароль AP
├── go.mod
└── templates/
    ├── index.html   # основная страница (Simple + Advanced)
    └── setup.html   # Setup Wizard
```

## 7. Функциональные требования

### 7.1 Онбординг (/setup)

- Доступен без авторизации только при `setup_done: false`
- Шаг 1: установка пароля панели (обязательно, мин. 6 символов)
- Шаг 2: установка пароля AP (опционально, мин. 8 символов, WPA2)
- Шаг 3: подтверждение и переход в панель
- После завершения: `setup_done: true`, редирект на `/login`

### 7.2 Аутентификация

- Логин + пароль, bcrypt хэш (cost=12)
- Session cookie TTL 24 часа, HttpOnly, SameSite=Strict
- Задержка 2 сек при неверном пароле

### 7.3 Simple Mode

- Статус VPN (индикатор + внешний IP)
- Дропдаун выбора активного сервера + кнопка переключить
- Текущая WiFi сеть + кнопка сменить
- Кнопки: перезапустить VPN, обновить подписку

### 7.4 Advanced Mode — Дашборд

- Статус сервисов: xray, redsocks, hostapd, dnsmasq, uap0
- Текущий внешний IP + локальный IP RPi
- Кнопка рестарта для каждого сервиса
- HTMX polling каждые 30 сек

### 7.5 Управление VLESS серверами (Advanced)

- Список серверов: имя, адрес, порт, активный/нет
- Добавление через вставку `vless://` ссылки — автопарсинг всех параметров
- Поддержка параметров: `address`, `port`, `uuid`, `flow`, `security`, `sni`, `pbk`, `sid`, `fp`
- Переключение → перезапись `config.json` + рестарт xray
- Удаление сервера

### 7.6 Подписки Remnawave (Advanced)

- HWID генерируется из MAC `wlan0` (SHA256, первые 16 символов)
- Заголовки при запросе: `x-hwid`, `x-device-os: Linux`, `x-device-model: RPi Zero W`
- HWID отображается в панели для регистрации в Remnawave
- Добавление URL подписки, автозагрузка и парсинг серверов
- Кнопка обновления подписки

### 7.7 WiFi

- **Simple**: текущая сеть, список доступных, подключение с паролем
- **Advanced**: дополнительно уровень сигнала и тип безопасности
- Запись в `/etc/wpa_supplicant/wpa_supplicant.conf` + `wpa_cli reconfigure`

### 7.8 Логи сервисов (Advanced)

- Последние 100 строк через `journalctl -u <service> -n 100`
- Обновление по кнопке
- Подсветка строк ERROR/WARNING

### 7.9 Настройки (Advanced)

- Смена пароля панели (требует текущий пароль)
- Смена пароля AP / сделать открытой (требует текущий пароль)

## 8. API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/setup` | Страница онбординга (только при `setup_done:false`) |
| POST | `/setup/password` | Установить пароль панели (шаг 1) |
| POST | `/setup/ap-password` | Установить пароль AP (шаг 2) |
| POST | `/setup/complete` | Завершить онбординг (шаг 3) |
| GET | `/` | Главная (редирект на `/setup` или `/login`) |
| POST | `/auth/login` | Авторизация |
| POST | `/auth/logout` | Выход |
| GET | `/api/status` | Статус сервисов + IP |
| POST | `/api/restart/{service}` | Рестарт сервиса |
| GET | `/api/vless` | Список VLESS серверов |
| POST | `/api/vless` | Добавить сервер (vless:// URL) |
| POST | `/api/vless/{id}/activate` | Активировать сервер |
| DELETE | `/api/vless/{id}` | Удалить сервер |
| POST | `/api/sub` | Загрузить sub-ссылку |
| GET | `/api/hwid` | HWID устройства |
| GET | `/api/wifi/scan` | Сканировать WiFi сети |
| GET | `/api/wifi/status` | Текущая WiFi сеть |
| POST | `/api/wifi/connect` | Подключиться к сети |
| GET | `/api/logs/{service}` | Логи сервиса |
| POST | `/api/settings/panel-password` | Сменить пароль панели |
| POST | `/api/settings/ap-password` | Сменить/убрать пароль AP |

## 9. Хранение данных

Все данные хранятся в `/etc/rpi-panel/`:

| Файл | Содержимое |
|------|------------|
| `config.json` | Логин, bcrypt-хэш пароля, `setup_done`, `ap_secured`, порт |
| `servers.json` | Список VLESS серверов, индекс активного |
| `subs.json` | URL подписок, дата последнего обновления |

Конфиг AP: `/etc/hostapd/hostapd.conf` — `wpa_passphrase` добавляется/удаляется при смене пароля AP.

Конфиг xray: `/usr/local/etc/xray/config.json` — перезаписывается при смене активного сервера.

## 10. Деплой

### 10.1 Разработка (macOS)

```bash
# Кросс-компиляция под RPi Zero W
GOOS=linux GOARCH=arm GOARM=6 go build -o rpi-panel ./...
```

Шаблоны и статика встроены в бинарник через `embed.FS`.

### 10.2 Деплой на RPi

```bash
scp rpi-panel max@stackplayer.local:~/rpi-panel
ssh rpi_local "sudo systemctl restart rpi-panel"
```

`make deploy` — автоматизирует оба шага.

### 10.3 Makefile

```makefile
build:
	GOOS=linux GOARCH=arm GOARM=6 go build -o rpi-panel ./...

deploy: build
	scp rpi-panel max@stackplayer.local:~/rpi-panel
	ssh rpi_local "sudo systemctl restart rpi-panel"
```

### 10.4 systemd сервис

```ini
[Unit]
Description=RPi Panel
After=network.target uap0.service

[Service]
User=root
ExecStart=/home/max/rpi-panel
Restart=always

[Install]
WantedBy=multi-user.target
```

## 11. Безопасность

- Пароль хранится как bcrypt хэш (cost=12)
- Session cookie: HttpOnly, SameSite=Strict, TTL 24ч
- Whitelist системных команд — только разрешённые сервисы
- Simple/Advanced режим — только UI, не влияет на права доступа к API
- Панель доступна только из LAN (нет проброса портов)
- Нет HTTPS в v1.0 (LAN-only, планируется в v1.1)
